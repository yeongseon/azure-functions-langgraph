# OpenAPI bridge example

Demonstrates wiring [`azure-functions-langgraph`](https://pypi.org/project/azure-functions-langgraph/) into [`azure-functions-openapi-python`](https://pypi.org/project/azure-functions-openapi-python/) via the `register_with_openapi` bridge.

`register_with_openapi(app)` reads the immutable metadata snapshot from `LangGraphApp.get_app_metadata()` and forwards each route (request/response models, summary, description, status codes) to `azure_functions_openapi.register_openapi_metadata()`. The result: every registered graph endpoint shows up in the OpenAPI spec generated by the openapi package, including request/response schemas when `request_model` / `response_model` are Pydantic models.

## Files

- `function_app.py` — registers `echo_agent`, then calls `register_with_openapi`
- `graph.py` — minimal echo graph
- `host.json`, `local.settings.json.example`, `requirements.txt`

## Run locally

```bash
cd examples/openapi_bridge
cp local.settings.json.example local.settings.json
pip install -r requirements.txt
func start
```

`register_with_openapi` returns the number of routes forwarded; the example prints it on startup so you can see the bridge fired before any HTTP traffic.

## Exposing the spec / Swagger UI

This bridge only forwards **metadata**. Exposing `/openapi.json` or Swagger UI is the responsibility of [`azure-functions-openapi-python`](https://github.com/yeongseon/azure-functions-openapi-python) — follow that package's own setup instructions, then point it at the same `function_app` instance:

```python
from azure_functions_openapi import setup_swagger  # see openapi package docs

setup_swagger(langgraph_app.function_app)  # exact API per openapi package version
```

The bridge requires `azure-functions-openapi-python >= 0.16.0` (programmatic `register_openapi_metadata` API).

## Requesting Pydantic-typed routes

Pass `request_model=` / `response_model=` to `app.register(...)` to surface full schemas in the OpenAPI document. The bridge raises `TypeError` if either is not a `pydantic.BaseModel` subclass.
